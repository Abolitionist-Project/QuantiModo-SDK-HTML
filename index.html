Query Parameter — This is the unique ID that QuantiModo uses to identify your application. Obtain a client id by emailing info@quantimo.do.
realm (required)
Query Parameter — Name of the realm representing the users of your distributed applications and services. A \"realm\" attribute MAY be included to indicate the scope of protection.
redirectUri (required)
Query Parameter — The redirect URI is the URL within your client application that will receive the OAuth2 credentials.
responseType (required)
Query Parameter — If the value is code, launches a Basic flow, requiring a POST to the token endpoint to obtain the tokens. If the value is token id_token or id_token token, launches an Implicit flow, requiring the use of Javascript at the redirect URI to retrieve tokens from the URI #fragment.
scope (required)
Query Parameter — Scopes include basic, readmeasurements, and writemeasurements. The \"basic\" scope allows you to read user info (displayname, email, etc). The \"readmeasurements\" scope allows one to read a user's data. The \"writemeasurements\" scope allows you to write user data. Separate multiple scopes by a space.
state (required)
Query Parameter — An opaque string that is round-tripped in the protocol; that is to say, it is returned as a URI parameter in the Basic flow, and in the URI

Path Parameter — Search query which may be an entire variable name or a fragment of one.
categoryName (optional)
Query Parameter — Filter variables by category name. The variable categories include Activity, Causes of Illness, Cognitive Performance, Conditions, Environment, Foods, Location, Miscellaneous, Mood, Nutrition, Physical Activity, Physique, Sleep, Social Interactions, Symptoms, Treatments, Vital Signs, and Work.
source (optional)
Query Parameter — Specify a data source name to only return variables from a specific data source.
limit (optional)
Query Parameter — The LIMIT is used to limit the number of results returned. So if you have 1000 results, but only want to the first 10, you would set this to 10 and offset to 0.
offset (optional)
Query Parameter — Now suppose you wanted to show results 11-20. You'd set the offset to 10 and the limit to 10.

array[Variable]
